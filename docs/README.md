# Documentation Index

## Folders

| Folder | Purpose |
|--------|---------|
| **api/** | API endpoint documentation, request/response formats |
| **architecture/** | System architecture, diagrams, component docs |
| **qa/** | QA test suites, test reports, checklists |
| **migration/** | Migration plans, step-by-step guides |
| **infrastructure/** | Server setup, Caddy config, DNS setup |

## How to Add Documentation

1. Create a markdown file in the appropriate folder
2. Use clear, descriptive filenames (e.g., `BACKEND-API-ENDPOINTS.md`)
3. Include a header with date and author
4. Commit and push changes

## Naming Convention

- Use UPPERCASE for main doc files
- Use hyphens between words
- Include date in reports (e.g., `QA-REPORT-2026-04-16.md`)
